Face Detection and Authentication

UIDAI is exploring possibilities to enable Face authentication on the desktop in a browser context. The proposed architecture to provide Face Authentication in the browser context requires AI on the edge to perform liveness checks of the face being captured by the webcam or connected camera. UIDAI is considering the injection of ML models using an appropriate binary code delivery mechanism. The security of these models is crucial for transaction integrity, and therefore an innovative solution is sought to protect the model from any tampering and reverse engineering.

As part of the challenge, participating teams are to demonstrate model security in a browser context using either obfuscation or cryptography. Models are typically 5~7MB in size and structured as flatbuffers. These models will be downloaded when first accessed on a desktop and then cached in the browser context. For subsequent face authentication transactions, a cached model is preferred unless the model has changed or updated. Teams are free to choose ONNX web runtime, Tensor.js, or any other innovative model to distribute the model in the browser context.

The expected outcome of this project is a functional solution that incorporates the security of the ML model at the edge. The backend for the proposed solution system must be designed to handle the high volume of transactions expected for any population-scale solutions.
